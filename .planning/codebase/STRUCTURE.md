# STRUCTURE.md — Directory Layout & File Organization

## Top-Level Structure

```
Oak/
├── Oak/                          # Main app source
│   ├── Models/                   # Data models, enums, protocols
│   ├── Views/                    # SwiftUI Views + window controller
│   ├── ViewModels/               # ObservableObject classes
│   ├── Services/                 # Business logic managers
│   ├── Extensions/               # Swift/AppKit extensions
│   ├── Resources/                # Assets (sounds, icons, config)
│   ├── OakApp.swift              # App entry point
│   ├── Oak.entitlements          # Sandbox entitlements
│   └── Info.plist                # Bundle metadata
├── Tests/
│   └── OakTests/                 # All unit tests
├── Oak.xcodeproj/                # Generated by XcodeGen
├── project.yml                   # XcodeGen project definition
├── .swiftlint.yml                # Lint configuration
├── .swiftformat                  # Format configuration
└── justfile                      # Task automation
```

## Naming Conventions

- **Directories**: PascalCase (`Models`, `Views`, `Services`)
- **Files**: PascalCase matching primary type (`FocusSessionViewModel.swift`)
- **Test files**: `{TypeName}Tests.swift` or `US{###}Tests.swift` (user story tests)
- **Extension files**: `{Type}+{Aspect}.swift` (`NSScreen+UUID.swift`)
- **View extension files**: `{ViewName}+{Aspect}.swift` (`NotchCompanionView+Controls.swift`)
